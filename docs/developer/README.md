Welcome! This landing page connects the workflows and standards that keep GeneaX running smoothly. Follow the sequence below
for a clean local setup, consistent configuration, and confident deployments.

## Orientation
- Review the project story and objectives in the [documentation overview](../README.md).
- Confirm domain context via the [master outline](../master-outline.md#master-outline) and [architecture](../core/architecture.md#geneax-architecture) guide.

## Build workflow
1. **Install prerequisites** using the [installation checklist](installation.md).
2. **Configure your environment** with help from the [configuration reference](configuration.md).
3. **Run the test suite** following [testing guidance](testing.md) before opening a pull request.
4. **Document changes** in the [changelog](changelog.md) and align them with the [roadmap](../core/roadmap.md).

## Collaboration standards
- Read the [contribution guidelines](contributing.md) before submitting patches.
- Consult the [deployment playbook](deployment.md) for staging and production promotion.
- Keep an eye on security expectations in the [security policy](../security.md).
